Проектирование БД
Основные таблицы
1. books — основная таблица книг.
- id (INTEGER PRIMARY KEY AUTOINCREMENT)
- title_ru (TEXT NOT NULL) — русское название
- title_orig (TEXT) — оригинальное название (может быть NULL)
- year_published (INTEGER) — год выпуска книги (издания, в котором она вышла)
- year_written (TEXT) — год(ы) написания текста (можно хранить как строку, например "1890-1892")
- description (TEXT) — краткое описание с обложки
- rating (INTEGER) — оценка от 1 до 10 (можно сделать CHECK)
- review (TEXT) — личное мнение/рецензия
- notes (TEXT) — заметки, цитаты
- cover_image (BLOB или TEXT) — картинка. Хранить как BLOB в базе не очень хорошо для производительности, лучше сохранять файл на диск и хранить путь. Для начала можно вообще пропустить это поле или хранить как TEXT с путём к файлу.
- Поля, связанные с процессом чтения:
	- status_id (INTEGER) — внешний ключ к таблице reading_statuses
	- start_date (DATE)
	- finish_date (DATE) (может быть NULL, если не завершена)
	- total_pages (INTEGER) — общее количество страниц (если бумажная/электронная)
	- total_duration_minutes (INTEGER) — общая длительность в минутах (если аудиокнига)
	- current_page (INTEGER) — текущая страница (если применимо)
	- current_position_minutes (INTEGER) — текущая позиция в минутах (если аудиокнига)
	- Важно: вид книги (бумажная, электронная, аудиокнига) определяет, какие из полей объёма и текущей позиции используются. Поэтому вид лучше вынести в отдельную таблицу.
	- Замечание по вычисляемому полю % прочитано: его логично вычислять в коде приложения на основе current_page/total_pages или current_position/total_duration, а не хранить в БД. Это избавит от проблем с синхронизацией.

2. authors — авторы.
- id (INTEGER PRIMARY KEY)
- name (TEXT UNIQUE NOT NULL) — имя автора
3. book_authors — связь многие-ко-многим (одна книга может иметь несколько авторов, один автор может написать несколько книг).
- book_id (INTEGER, FOREIGN KEY REFERENCES books(id) ON DELETE CASCADE)
- author_id (INTEGER, FOREIGN KEY REFERENCES authors(id) ON DELETE CASCADE)
- Первичный ключ — пара (book_id, author_id)
4. genres — жанры.
- id (INTEGER PRIMARY KEY)
- name (TEXT UNIQUE NOT NULL) — название жанра
5. book_genres — связь многие-ко-многим (книга может иметь несколько жанров).
- book_id (INTEGER, FOREIGN KEY REFERENCES books(id) ON DELETE CASCADE)
- genre_id (INTEGER, FOREIGN KEY REFERENCES genres(id) ON DELETE CASCADE)
- Первичный ключ — пара (book_id, genre_id)
6. languages — языки книги.
- id (INTEGER PRIMARY KEY)
- name (TEXT UNIQUE NOT NULL) — например, "русский", "английский", "японский"
7. book_types — виды книг (аудиокнига, бумажная, электронная).
- id (INTEGER PRIMARY KEY)
- name (TEXT UNIQUE NOT NULL)
Тогда в таблице books добавляем type_id (FOREIGN KEY).
8. reading_statuses — состояния чтения.
- id (INTEGER PRIMARY KEY)
- name (TEXT UNIQUE NOT NULL) — "Прочитано", "В процессе", "Брошена", "Не начата"
В таблице books будет status_id
